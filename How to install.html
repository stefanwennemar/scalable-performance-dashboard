Willkommen — los geht's mit deinem Dashboard.

Ein paar kleine Schritte. Beim ersten Mal ungefähr zehn Minuten, größtenteils Warten. Danach reicht ein Doppelklick auf Run Dashboard, um dein Portfolio im Browser zu sehen.

Vorbereitung — CSV aus der Scalable-App exportieren

A

Scalable-Capital-App auf deinem Handy öffnen

  1. Gehe zu Transaktionen.
  2. Tippe oben rechts auf den Drei-Punkte-Button (⋯).
  3. Tippe auf Alle auswählen.
  4. Tippe auf den Teilen-Button und wähle Transaktionen CSV.
  5. Speichere die Datei und übertrage sie auf deinen Computer (siehe unten).
B

Datei auf den Computer bringen

Nimm einfach den Weg, der für deine Handy-/Computer-Kombination am einfachsten ist.

iPhone → Mac   (am einfachsten)

  • AirDrop: Aus dem iOS-Teilen-Menü AirDrop und deinen Mac auswählen. Die CSV landet im Downloads-Ordner.

iPhone → Windows

  • E-Mail an dich selbst: Aus dem Teilen-Menü "Mail" wählen, an deine eigene Adresse schicken, am PC den Anhang herunterladen. Klappt immer.
  • iCloud Drive: Auf dem iPhone in Dateien → iCloud Drive speichern. Auf dem PC einmal iCloud für Windows installieren, dann taucht die Datei im Datei-Explorer auf.

Android → Mac oder Windows

  • Google Drive (oder eine andere Cloud): Aus dem Teilen-Menü hochladen, am Computer herunterladen.
  • E-Mail an dich selbst: Selbe Idee wie oben.
Wohin mit der Datei: Lege sie in den Ordner transaction_data direkt neben dieser Anleitung. Das Dashboard nimmt automatisch immer die neueste CSV aus diesem Ordner — alte Exporte kannst du also ruhig liegen lassen.

Wähle dein Betriebssystem

1

uv installieren (einmalig, ~30 Sekunden)

uv ist ein kleines Tool, das Python und die Bibliotheken für das Dashboard für dich installiert. Öffne das Terminal (drücke ⌘ + Leertaste, tippe Terminal und drücke Enter), füge dann diese eine Zeile ein und drücke nochmal Enter: 

curl -LsSf https://astral.sh/uv/install.sh | sh

Du siehst etwas Download-Fortschritt und am Ende eine Zeile wie "uv is installed". Danach kannst du das Terminal-Fenster schließen.

2

Setup ausführen

Doppelklick im Finder auf die Datei Setup.command in diesem Ordner.

Ein Terminal-Fenster öffnet sich und führt drei Schritte aus. Der erste Lauf dauert ungefähr drei Minuten — Python und die Bibliotheken werden geladen. Wenn "Setup complete!" erscheint, eine beliebige Taste drücken, fertig.

Wenn macOS sich weigert (zum Beispiel "kann nicht geöffnet werden"): Rechtsklick auf Setup.command, Öffnen wählen, im Dialog nochmal Öffnen klicken. Beim zweiten Mal funktioniert dann der ganz normale Doppelklick.
3

CSV hineinlegen und Dashboard starten

Lege die exportierte CSV (siehe oben) in den Ordner transaction_data (direkt neben dieser Anleitung).

Dann Doppelklick auf Run Dashboard.command. Dein Browser öffnet sich automatisch unter http://127.0.0.1:8050, sobald das Dashboard bereit ist.

  • Erster Start: 5–7 Minuten (Live-Preise werden geladen).
  • Folgende Starts: in der Regel unter einer Minute.
  • Zum Beenden einfach das Terminal-Fenster schließen.
4

Optional: Live-Daten direkt aus Scalable

Das Dashboard funktioniert auch komplett ohne dies. Wenn du möchtest, kann es aber Live-Kurse und tagesaktuelle Bewertungen direkt aus deinem Scalable-Konto holen — viel schneller als die gettex-Suche und ohne Lücken bei Derivaten. Du brauchst dafür Scalables offizielles Tool sc (Beta).

Einmalig installieren (im Terminal): 

brew tap scalablecapital/sc && brew install sc

Falls du Homebrew noch nicht hast, einmalig: 

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Einloggen. Klicke nach dem Start des Dashboards oben rechts auf die Schaltfläche Scalable API · connect und dann auf Connect now. Es öffnet sich automatisch ein Browser-Tab — dort einloggen und auf dem Handy in der Scalable-App bestätigen. Sobald das durch ist, schaltet das Dashboard von allein um. Die Pille oben rechts wird grün.

Es wird nichts hochgeladen — die Sitzung lebt nur auf deinem Computer. Die Verbindung kann jederzeit wieder gelöst werden mit sc logout.

Was passiert da eigentlich?

Deine CSV bleibt auf deinem Computer. Das Dashboard läuft als kleiner lokaler Webserver — die Adresse ist 127.0.0.1, also erreichbar ausschließlich von deinem eigenen Rechner. Live-Preise kommen von gettex.de; historische Kurse für deine Positionen und für den MSCI-ACWI-Benchmark holt das Dashboard von der gleichen Quelle. Es wird nichts hochgeladen.

Daten aktualisieren

Wenn du gehandelt hast, exportiere eine neue CSV (siehe oben) und lege sie in den transaction_data-Ordner. Das Dashboard nimmt automatisch die neueste Datei. Danach Doppelklick auf Run Dashboard.

Wenn etwas nicht klappt

  • Browser sagt "Verbindung nicht möglich" beim allerersten Start: einfach das Launcher-Fenster offen lassen und warten — der erste Lauf lädt rund 200 MB Kursdaten. Der Browser aktualisiert sich von selbst, sobald der Server bereit ist.
  • Zahlen sehen seltsam aus: die Live-Preise sind evtl. veraltet. Oben rechts im Dashboard auf Refresh prices klicken.
  • Neuer Laptop: kompletten Ordner kopieren und Setup auf der neuen Maschine erneut ausführen.

Gebaut mit Python, Dash, Plotly, gettex.de und yfinance. Kein Tracking, keine Telemetrie, kein Internet nötig nach der Einrichtung (außer für das Aktualisieren der Live-Preise).
